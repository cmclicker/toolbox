# Node.js — resources

> Curated external material for Node.js. My own explainers are in [../../references/node/](../../references/node/); TypeScript and JavaScript have their own catalogs in this cluster.

## Official docs & specs

- **[Node.js documentation](https://nodejs.org/api/)** — the API reference for every built-in module.
- **[Node.js Learn](https://nodejs.org/en/learn)** — official guides (event loop, async, modules, diagnostics).
- **[npm docs](https://docs.npmjs.com/)** — CLI commands, `package.json`, and registry behavior.
- **[Node releases & LTS schedule](https://nodejs.org/en/about/previous-releases)** — which versions are supported.

## Learn

- **[The Node.js event loop guide](https://nodejs.org/en/learn/asynchronous-work/event-loop-timers-and-nexttick)** — the authoritative explanation of micro/macrotasks and timers.
- **[Node.js Design Patterns (book)](https://www.nodejsdesignpatterns.com/)** — deeper architecture/patterns once the basics are solid.

## Cheatsheets & quick-reference

- **[Semver calculator](https://semver.npmjs.com/)** — paste a range, see exactly which versions it matches.
- **[node.green](https://node.green/)** — which JS/ES features are supported in which Node version.

## Tools & utilities

- **[nvm](https://github.com/nvm-sh/nvm)** · **[fnm](https://github.com/Schniz/fnm)** · **[Volta](https://volta.sh/)** — Node version managers ([../../references/node/versions-and-managers.md](../../references/node/versions-and-managers.md)).
- **[pnpm](https://pnpm.io/)** — fast, strict, disk-efficient package manager; strong for monorepos.
- **[publint](https://publint.dev/)** · **[Are the types wrong?](https://arethetypeswrong.github.io/)** — lint a package's publish config and type exports before shipping.
- **[npm-check-updates](https://github.com/raineorshine/npm-check-updates)** — bump `package.json` ranges to latest in a controlled pass.

## Best practices

- **[Node.js Best Practices (goldbergyoni)](https://github.com/goldbergyoni/nodebestpractices)** — a large, curated, citation-backed list covering structure, error handling, security, performance.
