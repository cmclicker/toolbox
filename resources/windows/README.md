# Windows — resources

> Curated external material for Windows as a dev environment. My own explainers are in [../../references/windows/](../../references/windows/). Shell-language resources: [../powershell/](../powershell/), [../bash/](../bash/).

## Official docs & specs

- **[Windows Dev environment docs](https://learn.microsoft.com/en-us/windows/dev-environment/)** — Microsoft's hub for setting up Windows (and WSL) for development.
- **[WSL documentation](https://learn.microsoft.com/en-us/windows/wsl/)** — install, interop, performance, and configuration.
- **[Windows Terminal documentation](https://learn.microsoft.com/en-us/windows/terminal/)** — profiles, settings, key bindings.
- **[winget documentation](https://learn.microsoft.com/en-us/windows/package-manager/winget/)** — every command and the manifest/export format.

## Learn

- **[Set up a dev environment on Windows](https://learn.microsoft.com/en-us/windows/dev-environment/overview)** — opinionated setup guides (WSL, VS Code, package managers) by stack.
- **[WSL best practices](https://learn.microsoft.com/en-us/windows/wsl/setup/environment)** — the filesystem-performance and VS Code/Git guidance distilled.

## Tools & utilities

- **[PowerToys](https://learn.microsoft.com/en-us/windows/powertoys/)** — power-user utilities (FancyZones, PowerToys Run, Keyboard Manager, file rename). Install via `winget install Microsoft.PowerToys`.
- **[Sysinternals](https://learn.microsoft.com/en-us/sysinternals/)** — deep system tools (Process Explorer, Process Monitor, Autoruns, Handle) for diagnosing what a process is really doing.
- **[Scoop](https://scoop.sh/)** · **[Chocolatey](https://chocolatey.org/)** — alternative package managers (see [../../references/windows/winget-and-package-management.md](../../references/windows/winget-and-package-management.md)).

- **[ss64 — PowerShell](https://ss64.com/ps/)** — the PowerShell companion.
