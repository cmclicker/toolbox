# Linux — resources

> Curated external material for Linux as a dev/server environment. My own explainers are in [../../references/linux/](../../references/linux/); the shell language is in [../bash/](../bash/).

## Official docs & specs

- **[man7.org — Linux man pages](https://man7.org/linux/man-pages/)** — the authoritative manual for syscalls, commands, and config formats, nicely rendered online.
- **[The Linux Documentation Project](https://tldp.org/)** — long-form guides (some dated, but the fundamentals hold).
- **[Filesystem Hierarchy Standard](https://refspecs.linuxfoundation.org/FHS_3.0/fhs/index.html)** — the spec behind [filesystem-hierarchy.md](../../references/linux/filesystem-hierarchy.md).

## Learn

- **[The Missing Semester (MIT)](https://missing.csail.mit.edu/)** — the shell, command-line tools, and dev environment skills courses skip. Excellent.
- **[Linux Journey](https://linuxjourney.com/)** — structured beginner→intermediate path through the OS.
- **[Arch Wiki](https://wiki.archlinux.org/)** — despite the name, the best practical reference for systemd, networking, and configuration on *any* distro.

## Cheatsheets & quick-reference

- **[ss64 — Bash/Linux](https://ss64.com/bash/)** — fast per-command reference.
- **[tldr pages](https://tldr.sh/)** — example-first summaries (`tldr tar` beats reading the man page for common usage).
- **[devhints](https://devhints.io/)** — one-page references for many tools.

## Tools & utilities

- **[explainshell](https://explainshell.com/)** — paste a command line; it annotates each piece from the man pages.
- **[ripgrep (`rg`)](https://github.com/BurntSushi/ripgrep)** · **[fd](https://github.com/sharkdp/fd)** · **[bat](https://github.com/sharkdp/bat)** — faster, friendlier modern replacements for grep/find/cat.
- **[htop](https://htop.dev/)** / **[btop](https://github.com/aristocratos/btop)** — interactive process/resource monitors.
